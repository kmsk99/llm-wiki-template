# Query: 질문 → 답변 → 위키 재기록

위키에 질문하고 답변을 받되, 답변 과정에서 발견한 통찰을 다시 위키에 기록하는 선순환 루프.

## 입력

$ARGUMENTS

자연어 질문을 입력한다. 예:
- `사장링크 시스템의 데이터 수집 주기는?`
- `브랜더스의 사업자등록 정보`
- `온보딩 때 신규 입사자가 먼저 읽어야 할 문서는?`

## 실행 절차

### Phase 1: 검색 (Graphify 우선)

**원칙: 의미는 Graphify, 문자는 Grep.** 주제·관계·구조 탐색은 항상 Graphify로 시작한다.

1. 질문의 키워드와 핵심 개념을 파악한다.
2. **Graphify 지식 그래프를 1차 탐색한다:**
   - `graphify-out/GRAPH_REPORT.md`의 Community Hubs, God Nodes를 훑어 질문이 어느 커뮤니티/노드에 속하는지 판단한다.
   - 의미 기반 질의: `/graphify query "<질문>"` 또는 `graphify query "<질문>"`
   - 두 개념 간 연결 질문이면: `/graphify path "<A>" "<B>"`
   - 특정 개념의 주변 탐색이면: `/graphify explain "<개념>"`
   - **그래프가 stale하면** (wiki가 GRAPH_REPORT.md 타임스탬프 이후로 바뀐 경우) 먼저 `graphify . --update` 후 재질의한다.
3. Graphify가 지목한 wiki 문서를 실제로 읽는다.
4. **보조 검색 (필요 시에만):** 정확한 문자열·숫자·파일 경로 확인이 필요하면 Grep/Glob 사용. tags/파일명/index도 보조 입력으로 활용.

### Phase 2: 답변
4. 위키에 있는 정보만으로 답변을 구성한다.
5. 답변에 사용한 wiki 페이지를 인용한다:
   ```
   사장링크의 데이터 수집은 일 1회 수행됩니다.
   — 출처: [wiki/systems/sajang-link.md]
   ```
6. 위키에 정보가 부족하면 솔직하게 밝힌다:
   ```
   ⚠️ 이 질문에 대한 정보가 wiki에 부족합니다.
   관련될 수 있는 raw 소스: raw/meetings/2026-04-06-topic.md
   ```

### Phase 3: 위키 강화 (Query가 RAG와 다른 핵심)

답변 과정에서 다음을 발견하면 wiki를 갱신한다.
**단, 수정은 답변 과정에서 실질적 발견이 있을 때만 한다.** 단순 사실 조회 질문(예: "사업자등록번호는?")에는 wiki를 수정하지 않는다. 수정 기준:
- cross-link 2개 이상 누락이 확인될 때
- 명백한 지식 갭이나 오류가 발견될 때
- 질문이 기존 문서의 불명확한 서술을 드러낼 때

7. **새로운 연결**: 서로 관련되지만 링크가 없던 문서 간 cross-link 추가
8. **누락된 정보**: 질문은 합리적인데 wiki에 답이 없으면:
   - 관련 raw 소스가 있으면 → `TODO: ingest 필요` 표시를 해당 문서에 추가
   - raw에도 없으면 → 해당 주제의 wiki 문서에 `Open question` 추가
9. **불명확한 설명**: 답변하면서 기존 문서의 설명이 불명확하다고 느끼면 개선
10. **색인 갭**: 질문 주제가 index에 없으면 관련 index 갱신

### Phase 4: 보고
11. 답변 출력 후, 위키 변경 사항을 요약한다:

```
---
### 📝 Wiki 갱신 사항
- wiki/systems/sajang-link.md: cross-link 2개 추가
- wiki/index/data-collection.md: 색인에 항목 추가
- wiki/processes/data-pipeline.md: Open question 1개 추가
```

## 규칙
- 위키에 없는 내용을 있는 것처럼 답하지 않는다.
- 답변의 모든 사실에 출처를 명시한다.
- 위키 갱신은 답변과 별도 섹션으로 보고한다. 갱신이 없으면 해당 섹션을 생략한다.
- 대규모 갱신이 필요하면 직접 수행하지 말고 `/project:ingest`를 권장한다.
- 질문에 답하는 것이 주 목적이다. 위키 강화는 부수 효과이지 목표가 아니다.
