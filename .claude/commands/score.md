# Score: 신뢰도 평가

wiki 문서의 사실들에 신뢰도를 평가하고, 전체 위키의 신뢰도 현황을 관리한다.

## 입력

$ARGUMENTS

- 인자가 없으면: wiki/ 전체 문서의 신뢰도를 평가한다.
- 파일 경로가 주어지면: 해당 문서만 평가한다.
- `--summary`가 붙으면: 전체 신뢰도 현황 요약만 출력한다.

## 신뢰도 등급

| 등급 | 의미 | 기준 |
|------|------|------|
| 🟢 High | 검증됨 | 1차 소스(공식 문서, 계약서, 시스템 데이터)에서 직접 확인 |
| 🟡 Medium | 합리적 | 회의록, 슬랙 대화 등 2차 소스에서 확인. 해석 여지 있음 |
| 🟠 Low | 추정 | 간접 정보, 단일 소스, 오래된 정보 |
| 🔴 Unverified | 미검증 | 출처 없음, TODO/Open question 표시됨 |

## 평가 기준

각 문서의 핵심 사실(Key facts)에 대해 다음을 점검한다:

1. **소스 존재 여부**: frontmatter `source:`가 실제 raw 파일을 가리키는가?
2. **소스 품질**: 1차 소스인가, 전해들은 것인가?
3. **최신성**: `updated:` 날짜가 현재 날짜 기준 30일 이내인가?
4. **교차 검증**: 같은 사실이 2개 이상 소스에서 확인되는가?
5. **모순 여부**: contradictions.md에 관련 항목이 있는가?
6. **마커 존재**: `TODO`, `Open question`, `Assumption`, `TBD`가 있는가?

## 실행 절차

1. 대상 문서를 읽는다.
2. 문서 내 핵심 사실(Key facts 섹션 또는 주요 주장)을 식별한다.
3. 각 사실에 대해 위 6가지 기준을 적용한다.
4. 문서 전체 신뢰도를 산출한다 (가장 낮은 등급 기준).
5. frontmatter에 `confidence:` 필드를 추가/갱신한다 (표준 스키마 확장 필드):
   ```yaml
   confidence: medium  # 2026-04-09 scored
   ```
   - 이 필드는 CLAUDE.md의 기본 frontmatter 스키마(`title`, `owner`, `status`, `updated`, `tags`, `source`)에 추가되는 선택적 확장 필드이다.
   - 기존 frontmatter 필드를 변경하지 않고 `confidence:`만 추가/갱신한다.

## 출력

```
## Confidence Score Report

### 문서별 신뢰도
| 문서 | 등급 | 주요 이유 |
|------|------|-----------|

### 등급 분포
- 🟢 High: N문서
- 🟡 Medium: N문서
- 🟠 Low: N문서
- 🔴 Unverified: N문서

### 신뢰도 향상 권장 사항
| 문서 | 현재 | 목표 | 필요한 조치 |
|------|------|------|-------------|
```

## 규칙
- 신뢰도 평가 자체가 사실을 변경하지 않는다 (읽기 전용 + confidence 필드만 추가).
- 평가 근거를 항상 명시한다.
- `--summary` 모드에서도 🔴 Unverified 문서는 반드시 나열한다.
