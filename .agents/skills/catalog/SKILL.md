# Catalog: Raw 소스 등록

새 raw 소스를 manifest에 등록하고 인제스트 준비를 한다.

## 입력

$ARGUMENTS

- 파일 경로가 주어지면: 해당 파일을 등록한다.
- 디렉토리가 주어지면: 디렉토리 내 모든 파일을 등록한다.
- 인자가 없으면: raw/ 전체를 스캔하여 manifest에 없는 파일을 찾아 등록한다.

## 실행 절차

1. 대상 파일을 식별한다.
2. `raw/.manifest.md`를 읽는다.
3. 이미 등록된 파일은 건너뛴다.
4. 새 파일에 대해 다음 메타데이터를 수집한다 (실제 manifest 테이블 컬럼 기준):
   - **파일**: `meetings/2026-04-09-회의.md` 형식의 상대 경로 (`raw/` 기준)
   - **유형**: 회의록 | 정제 메모 | 브리프 | 슬랙 | 전사 | 초안 | 링크 | 문서 | 스프레드시트 | 프레젠테이션 | 이미지 | 기타
   - **날짜**: 파일명에서 추출하거나, 파일 메타데이터에서 추출
   - **인제스트 상태**: `미정`
   - **비고**: 파일 내용을 간단히 설명
5. `raw/.manifest.md`에 항목을 추가한다. 기존 테이블 형식에 맞춘다:
   `| 파일경로 | 유형 | 날짜 | 인제스트 상태 | 비고 |`
6. 비텍스트 파일(PDF, XLSX, DOCX, PPTX 등)이면:
   - `scripts/parse-raw.sh <파일>`로 Marker 파싱을 실행한다.
   - 파싱 성공 시 비고에 `parsed` 표시를 추가한다.

## 출력

```
## Catalog Report

### 새로 등록된 소스
| 파일 | 유형 | 날짜 | 상태 |
|------|------|------|------|

### 이미 등록됨 (건너뜀)
- ...

### 파싱 결과
| 파일 | 파싱 결과 |
|------|-----------|
```

## 규칙
- raw/ 파일은 절대 수정하지 않는다.
- manifest 테이블 형식을 기존과 동일하게 유지한다: `| 파일 | 유형 | 날짜 | 인제스트 상태 | 비고 |`
- 다음 파일은 manifest에 등록하지 않는다:
  - `.parsed.md` 파일 (파싱 파생물)
  - Marker가 생성한 이미지 디렉토리 (예: `260204_사업자등록증명(브랜더스)/`)
  - `.DS_Store`, `.gitkeep`, `README.md` 등 시스템/안내 파일
- 인제스트 상태는 반드시 `완료`, `미정`, `부분`, `보류` 중 하나를 사용한다.
